# 五子棋项目重构 - 需求文档

## 1. 项目概述

本项目是一个在线五子棋对弈平台，支持多种专业五子棋规则，提供实时对战、等级分系统、观战等功能。

## 2. 数据库结构（必须兼容现有）

### 2.1 用户表 player
- `id`: 用户ID，主键
- `email`: 邮箱，登录用
- `nickname`: 昵称
- `password`: MD5加密密码
- `login_times`: 登录次数
- `b_win`: 执黑胜场
- `b_lose`: 执黑负场
- `w_win`: 执白胜场
- `w_lose`: 执白负场
- `draw`: 和棋数
- `games`: 总对局数
- `reg_time`: 注册时间
- `reg_ip`: 注册IP
- `last_login_time`: 最后登录时间
- `last_login_ip`: 最后登录IP
- `score`: ELO等级分(decimal 15,3)
- `vip`: VIP标识
- `language`: 语言设置(默认zh-CN)
- `intro`: 个人简介

### 2.2 对局表 games
- `id`: 对局ID，主键
- `black_id`: 黑方用户ID
- `white_id`: 白方用户ID
- `status`: 状态(0未开始,1进行中,2黑胜,4白胜,8和棋)
- `is_private`: 是否私密对局
- `offer_draw`: 提和用户ID
- `rule`: 规则类型(Yamaguchi,RIF,Soosyrv8,Renju,Gomoku,TaraGuchi)
- `free_opening`: 是否自由开局
- `allow_undo`: 是否允许悔棋
- `allow_ob_talk`: 是否允许旁观说话
- `game_record`: 棋谱记录(坐标字符串，如887868...)
- `black_time`: 黑方剩余秒数
- `white_time`: 白方剩余秒数
- `totaltime`: 总时间(秒)
- `step_add_sec`: 每步加秒
- `swap`: 交换标记
- `soosyrv_swap`: 索索夫规则交换标记
- `a5_pos`: 五手打点位置
- `a5_numbers`: 五手打点数
- `updtime`: 更新时间
- `movetime`: 最后落子时间
- `comment`: 对局备注
- `tid`: 比赛ID
- `vip`: VIP对局标记
- `create_time`: 创建时间

### 2.3 对局邀请表 game_invites
- `id`: 邀请ID
- `from`: 发起者ID
- `to`: 接收者ID
- `black_id`: 指定黑方ID
- `message`: 邀请消息
- `totaltime`: 总时间
- `step_add_sec`: 每步加秒
- `rule`: 规则
- `free_opening`: 是否自由开局
- `allow_undo`: 是否允许悔棋
- `allow_ob_talk`: 是否允许旁观说话
- `status`: 状态(0等待,1已同意,-1已失效)
- `is_private`: 是否私密
- `game_id`: 关联对局ID
- `updtime`: 更新时间

### 2.4 悔棋记录表 game_undo_log
- `id`: 记录ID
- `game_id`: 对局ID
- `uid`: 申请用户ID
- `current_board`: 申请时盘面
- `to_number`: 悔棋到第几手
- `comment`: 备注
- `status`: 状态(0待确认,1已同意,-1已拒绝)
- `created_time`: 创建时间

### 2.5 等级分变动记录表 score_log
- `id`: 记录ID
- `game_id`: 对局ID
- `player_id`: 玩家ID
- `op_id`: 对手ID
- `before_score`: 赛前分数
- `op_score`: 对手分数
- `k_val`: K值
- `delta_score`: 分数变动
- `after_score`: 赛后分数
- `logtime`: 记录时间

## 3. 功能需求详细说明

### 3.1 用户系统

#### 3.1.1 注册与登录
- 用户通过邮箱注册
- 密码使用MD5加密存储
- 登录后生成JWT Token
- 支持记住登录状态
- 记录登录IP和时间

#### 3.1.2 用户信息
- 查看和编辑个人资料
- 查看对局统计(执黑/执白胜负、和棋数)
- 查看等级分历史
- 设置语言偏好

#### 3.1.3 排行榜
- 全局等级分排名
- 分页展示(每页100人)
- 只显示有对局记录的玩家

### 3.2 对局邀请与协商

#### 3.2.1 发起邀请
- 从大厅点击用户发起邀请
- 设置对局参数:
  - 规则类型(Yamaguchi/RIF/Soosyrv8/Renju/Gomoku/TaraGuchi)
  - 总时间(分钟)
  - 每步加秒(0-90秒)
  - 是否执黑
  - 是否自由开局
  - 是否允许悔棋
  - 是否私密对局
  - 是否允许旁观说话
  - 邀请消息

#### 3.2.2 邀请协商
- 接收方可修改参数后回传
- 15秒内无响应进入协商模式
- 双方参数一致后开始对局

#### 3.2.3 接受邀请
- 创建对局记录
- 双方进入游戏房间
- 游戏开始

### 3.3 对局功能

#### 3.3.1 棋盘与落子
- 15x15标准棋盘
- 显示坐标标记
- 落子显示手数
- 最后落子高亮
- 悔棋后重新显示棋盘上棋子

#### 3.3.2 规则相关功能
- **自由开局**: 无限制 vs 标准开局限制
- **交换**: 在允许的阶段执行黑白交换
- **设置打点数**: Yamaguchi/Soosyrv8规则下指定5手打点数
- **打点落子**: 在指定位置落下打点标记
- **选择第五手**: 从打点中选择作为正式第5手

#### 3.3.3 对局操作
- **认输**: 立即结束对局，对方获胜
- **提和**: 向对方提出和棋请求
- **回应和棋**: 同意则和棋，拒绝则继续
- **申请悔棋**: 请求悔棋到指定手数(至少保留前5手，避免悔棋涉及到交换和打点等逻辑)
- **回应悔棋**: 同意则回退到指定手数，拒绝则继续

#### 3.3.4 时间控制
- 显示双方剩余时间
- 倒计时显示
- 超时自动判负
- 悔棋接受后给予提出方10%时间补偿

#### 3.3.5 聊天与表情
- 对局双方聊天
- 旁观者聊天(可开关)
- 发送表情(前端解析显示为图片)
- 发送棋盘推演图(在聊天中展示特定局面)
- 历史消息查看

#### 3.3.6 私密对局
- 仅对局双方可进入房间
- 旁观者无法观看
- 对局结束后可在历史记录中查看

### 3.4 对局结束与复盘

#### 3.4.1 胜负判定
- 五连获胜
- 禁手判负(有禁手规则)
- 超时判负
- 认输
- 和棋(双方同意)

#### 3.4.2 复盘功能
- 完整棋谱回放
- 前进/后退查看每一步
- 跳转到指定手数
- 查看悔棋历史(显示所有被悔掉的步骤)
- 显示对局信息(双方、规则、结果等)

### 3.5 大厅功能

#### 3.5.1 对局列表
- 显示进行中的对局
- 显示最近结束的对局(10分钟内)
- 显示对局ID、双方昵称、手数、规则
- 点击进入观战(私密对局除外)

#### 3.5.2 在线用户
- 显示大厅在线用户
- 点击用户发起邀请
- 显示用户等级分

#### 3.5.3 聊天
- 大厅公共聊天
- 表情支持

## 4. ELO等级分系统

### 4.1 计算公式
```
E = 1 / (1 + 10^((Rb - Ra) / 400))
Sa = Ra + K * (W - E)
```

### 4.2 K值规则
- 新用户(对局数<50)：K=30
- 普通用户(对局数>=50)：K=16
- 高手(分数>=2400)：K=10
- 惩罚规则：
  - 对局手数<5：K=1
  - 对局手数<16：K=5

### 4.3 记录内容
记录到score_log表：赛前分、对手分、K值、分差、赛后分。

## 5. 重构要求补充

### 5.1 多语言支持
- 服务端所有返回文本使用i18n key
- 前端/客户端根据用户设置加载对应语言包
- 支持中文、英文，可扩展其他语言

### 5.2 规则独立实现
- 每种规则独立实现，不共用逻辑
- 规则配置化，便于添加新规则
- 规则之间无依赖关系

### 5.3 前后端分离
- 服务端提供RESTful API
- WebSocket用于实时通信
- 客户端可以是Web、移动端APP、小程序等
- 统一接口，不区分客户端类型
