# ELO 等级分算法

## 1. 概述

本系统采用经典的 **ELO 评分系统** 计算玩家对局后的积分变化。ELO 系统最初用于国际象棋评分，现广泛应用于各类竞技游戏的匹配和排名系统。

---

## 2. 核心公式

### 2.1 期望胜率计算

```
        1
E = ---------------
    1 + 10^((Rb - Ra) / 400)
```

**参数说明**：
- `E`：玩家 A 的期望胜率（0 ~ 1 之间）
- `Ra`：玩家 A 的当前积分
- `Rb`：玩家 B 的当前积分
- `400`：ELO 系统标准分差常数

### 2.2 积分更新公式

```
R'a = Ra + K × (Sa - E)
```

**参数说明**：
- `R'a`：玩家 A 战后的新积分
- `Ra`：玩家 A 战前的积分
- `K`：K 值（积分变动系数）
- `Sa`：实际比赛结果（胜=1，负=0，平=0.5）
- `E`：期望胜率

---

## 3. 系统实现

### 3.1 代码实现

```php
// 参数初始化
$before_black_score = $black_player->score;   // 黑方战前积分
$before_white_score = $white_player->score;  // 白方战前积分

// 计算黑方期望胜率 (E)
// 白方积分 - 黑方积分，差值越小 E 越接近 0.5
$we = 1 / (1 + pow(10, (($before_white_score - $before_black_score) / 400)));

// 计算黑方积分变化
// result: 1=黑胜, 0=白胜, 0.5=平局
$delta_black = $k_black * ($result - $we);
```

### 3.2 K 值规则

K 值决定每局比赛的积分波动幅度。本系统采用**动态 K 值**：

| 条件 | K 值 | 说明 |
|------|------|------|
| 总对局数 < 50 | **30** | 新手保护期，积分波动大 |
| 总对局数 >= 50 | **16** | 正常玩家 |
| 积分 >= 2400 | **10** | 高手区 |
| 手数 < 5 局 | **1** | 惩罚：短对局不计分 |

```php
$k_black = 30;
$k_white = 30;

// 对局数达到50局后降低K值
if ($black_player->games >= 50) {
    $k_black = 16;
}
if ($white_player->games >= 50) {
    $k_white = 16;
}

// 高手降低K值
if ($black_player->score >= 2400) {
    $k_black = 10;
}
if ($white_player->score >= 2400) {
    $k_white = 10;
}

// 短对局惩罚
if ($moves < 5) {
    $k_black = 1;
    $k_white = 1;
}
```

### 3.3 结果处理

| 比赛结果 | result 值 | 黑方 delta | 白方 delta |
|----------|-----------|------------|------------|
| 黑胜 | 1 | `K × (1 - E)` | `-K × (1 - E)` |
| 白胜 | 0 | `-K × E` | `K × E` |
| 平局 | 0.5 | `K × (0.5 - E)` | `-K × (0.5 - E)` |

## 4. 积分变化示例

### 4.1 示例 1：势均力敌

假设：
- 黑方积分：2000
- 白方积分：2000
- K 值：30
- 黑胜

计算：
```
E = 1 / (1 + 10^((2000 - 2000) / 400))
  = 1 / (1 + 10^0)
  = 1 / 2
  = 0.5

delta_black = 30 × (1 - 0.5)
            = 30 × 0.5
            = +15

delta_white = -15
```

结果：黑方 +15 分，白方 -15 分

### 4.2 示例 2：实力悬殊

假设：
- 黑方积分：2400（高手）
- 白方积分：1600（新手）
- K 值：黑方=10，白方=30
- 黑胜

计算：
```
E = 1 / (1 + 10^((1600 - 2400) / 400))
  = 1 / (1 + 10^(-2))
  = 1 / (1 + 0.01)
  ≈ 0.99

delta_black = 10 × (1 - 0.99)
            = 10 × 0.01
            = +0.1  (≈ 0)

delta_white = -30 × 0.01
            = -0.3  (≈ 0)
```

结果：高手胜新手积分变化极小，避免"炸鱼"影响新手体验

### 4.3 示例 3：爆冷门

假设：
- 黑方积分：2400
- 白方积分：1600
- K 值：黑方=10，白方=30
- 白胜（爆冷）

计算：
```
E = 0.99（如上）

delta_black = 10 × (0 - 0.99)
            = -9.9  (≈ -10)

delta_white = 30 × (1 - 0.99)
            = 30 × 0.01
            = +0.3  (≈ 0)
```

结果：新手爆冷胜高分玩家，几乎不影响自己分数，但会大幅降低高分玩家分数

## 5. K 值影响分析

### 5.1 K 值与积分波动

```
K = 30：每局最多 ±30 分（新手期）
K = 16：每局最多 ±16 分（成长期）
K = 10：每局最多 ±10 分（高手区）
K = 1：  每局最多 ±1  分（短对局惩罚）
```

### 5.2 积分差距与期望胜率

| 积分差 (Rb - Ra) | 期望胜率 E |
|------------------|------------|
| 0 | 50% |
| 50 | 54.6% |
| 100 | 59.1% |
| 200 | 67.0% |
| 300 | 74.6% |
| 400 | 81.2% |
| 500 | 86.6% |
| 800 | 96.1% |
| 1000 | 98.6% |

> 当双方积分差达到 400 分时，弱者仅有约 19% 的胜率

## 7. 重构建议

### 7.1 核心接口设计

```typescript
interface EloRatingSystem {
    // 计算期望胜率
    calculateExpectedScore(ratingA: number, ratingB: number): number;
    
    // 计算积分变化
    calculateRatingChange(
        rating: number,
        expectedScore: number,
        actualScore: number,
        kValue: number
    ): number;
    
    // 获取动态 K 值
    getKValue(gamesPlayed: number, currentRating: number): number;
}
```

### 7.2 需保留的业务逻辑

1. **动态 K 值**：根据对局数和积分调整
2. **新手保护**：50 局内 K=30
3. **高分惩罚**：2400+ K=10
4. **短对局惩罚**：<5 手 K=1
5. **积分日志**：每局完整记录

### 7.3 建议的扩展

| 功能 | 说明 |
|------|------|
| 连胜奖励 | 可考虑增加连胜时的 K 值加成 |
| 段位系统 | 基于积分划分段位（新手/业余/专业等） |
| 赛季重置 | 每个赛季结束后衰减部分积分 |
