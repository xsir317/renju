# ELO等级分系统设计

## 1. 概述

本项目采用ELO等级分系统来评估玩家实力，根据对局结果动态调整玩家等级分。

## 2. 计算公式

### 2.1 基本公式

```
E = 1 / (1 + 10^((Rb - Ra) / 400))
Sa = Ra + K * (W - E)
```

其中：
- **Ra**: 玩家A的当前等级分
- **Rb**: 玩家B（对手）的当前等级分
- **E**: 玩家A的预期胜率（0到1之间）
- **W**: 实际结果（1=胜, 0.5=和, 0=负）
- **K**: 等级分变动系数
- **Sa**: 玩家A的新等级分

### 2.2 计算示例

假设：
- 玩家A等级分: 1800
- 玩家B等级分: 2000
- 结果: 玩家A获胜

计算过程：
```
E = 1 / (1 + 10^((2000 - 1800) / 400))
  = 1 / (1 + 10^(200 / 400))
  = 1 / (1 + 10^0.5)
  = 1 / (1 + 3.162)
  = 1 / 4.162
  = 0.240

假设 K = 16:
Sa = 1800 + 16 * (1 - 0.240)
   = 1800 + 16 * 0.760
   = 1800 + 12.16
   = 1812.16
```

## 3. K值规则

K值决定了等级分变动的敏感度，根据玩家情况动态调整：

### 3.1 基础K值

| 条件 | K值 | 说明 |
|-----|-----|------|
| 对局数 < 50 | 30 | 新玩家，快速定位实力 |
| 对局数 >= 50 且 分数 < 2400 | 16 | 普通玩家，稳定变动 |
| 分数 >= 2400 | 10 | 高手，精确调整 |

### 3.2 惩罚性K值调整

对于对局手数过少的情况，降低K值防止异常波动：

| 对局手数 | K值调整 |
|---------|---------|
| < 5手 | K = 1 (最小值) |
| < 16手 | K = 5 (若基础K值>5) |

说明：
- 对局手数过少时，结果可能带有较大随机性，不应让等级分大幅波动
- 如果双方同意和棋，不受此惩罚规则限制

### 3.3 K值计算示例

假设某玩家：
- 总对局数：45（< 50，基础K = 30）
- 当前等级分：1850（< 2400）
- 当前对局手数：12手（< 16，触发惩罚）

实际K值 = 5（按<16手的规则）

## 4. 数据库记录

### 4.1 记录内容

每场对局结束后，向 `score_log` 表插入两条记录（双方各一条）：

```sql
INSERT INTO score_log (
  game_id,        -- 对局ID
  player_id,      -- 玩家ID
  op_id,          -- 对手ID
  before_score,   -- 赛前分数
  op_score,       -- 对手分数
  k_val,          -- K值
  delta_score,    -- 分数变动
  after_score,    -- 赛后分数
  logtime         -- 时间
) VALUES (...);
```

### 4.2 记录示例

对局ID：12345
- 玩家A（ID: 1001）：赛前 1800分，赛后 1812.16分
- 玩家B（ID: 1002）：赛前 2000分，赛后 1987.84分

生成的记录：

| 字段 | 记录1 (玩家A) | 记录2 (玩家B) |
|-----|--------------|--------------|
| game_id | 12345 | 12345 |
| player_id | 1001 | 1002 |
| op_id | 1002 | 1001 |
| before_score | 1800.000 | 2000.000 |
| op_score | 2000.000 | 1800.000 |
| k_val | 16 | 16 |
| delta_score | +12.160 | -12.160 |
| after_score | 1812.160 | 1987.840 |
| logtime | 2026-02-27 12:00:00 | 2026-02-27 12:00:00 |

## 5. 等级分展示

### 5.1 用户信息显示

在用户资料中显示：
- 当前等级分（保留3位小数）
- 等级分排名（全局）
- 等级分历史走势图

### 5.2 排行榜

显示前100名玩家：
- 排名
- 昵称
- 等级分
- 总对局数
- 胜率

## 6. 特殊处理

### 6.1 和棋处理

和棋时，双方等级分变动较小：
- W = 0.5
- 双方得失分数相等，方向相反

### 6.2 新手保护

对局数 < 10 的玩家：
- 等级分不显示在排行榜
- 对局不计入对方K值计算（避免老手刷分）

### 6.3 异常对局

检测到异常的对局（如双方共谋）：
- 标记为无效对局
- 不计算等级分变动
- 保留记录供管理员审查
